# Secret schema ŌĆö `infra-bootstrap` Infisical project

The single source of truth for what lives in the `infra-bootstrap` Infisical project
(env `dev`). Referenced by `pigsty.yml` and by kubespray `group_vars/`.

All secrets live at the **project root** ŌĆö no folders. Folders were tried
and dropped: they add friction to every `infisical` CLI call for no benefit
at this project's size. One practical consequence: `infisical run` /
`infisical export` / `infisical secrets get` never need `--path` or
`--recursive` ŌĆö root is the only level, so the default already reaches
everything.

## Naming convention

- Secret name: `SCREAMING_SNAKE_CASE`
- Type: always `shared` (machine identity is project-scoped, no personal secrets)
- Environment: secrets live in `dev`; `prod` is created by duplicating the `dev` tree when the cluster goes live

## SSH access (per-host)

General-purpose SSH connection info for agent-driven ops against each host
ŌĆö per `DECISION.md`'s "Operations live on Hermes" model (an agent SSHes in
directly rather than the operator running commands by hand). Each host gets
a `_HOST` / `_KEY` / `_PORT` / `_USER` quartet, same naming pattern as the
other secrets above, all at project root:

| Secret group | Host | Purpose |
| --- | --- | --- |
| `SSH_PROXMOX_HOST/KEY/PORT/USER` | `.165` | primary, currently-live PVE host ŌĆö day-to-day PVE/VM/LXC ops outside what the Terraform token covers |
| `SSH_SERVER1_HOST/KEY/PORT/USER` | `192.168.1.200` | `server1` ŌĆö currently Debian 12 + libvirt, pending PVE reinstall; also hosts pg02's target HDD for pgBackRest |
| `SSH_LAPTOP_HOST/KEY/PORT/USER` | `192.168.1.161` | `ex-laptop` ŌĆö currently Debian 12 + libvirt, pending PVE reinstall as the 3rd node ([ADR-0013](adr/0013-pve-node-161-sleep-risk-mitigation.md)) |
| `SSH_PI4_HOST/KEY/PORT/USER` | `192.168.1.55` | Pi 4 ŌĆö Pi-hole/DNS helper, authoritative for `bnei.lan` |
| `SSH_OLDPG_HOST/KEY/PORT/USER` | `192.168.1.193` | old Pigsty Postgres setup, superseded by the new `pg-proxmox` cluster ŌĆö kept running as a rollback target in case of a major outage during/after migration, not yet decommissioned |

Generated by: user (manual, per-host keypair + connection details).
Consumed by: agent SSH sessions (Hermes) ŌĆö no dedicated wrapper script yet,
unlike `PVE_SSH_PRIVATE_KEY` which is Terraform-specific.

## Generation rules

- **Agent-generates-random** (32 bytes, base64): all `*_PASSWORD` and `*_TOKEN` rows above
  - Generator: `openssl rand -base64 32` or `head -c 32 /dev/urandom | base64`
  - Written via `infisical secrets set --projectId="$PROJ" --env=dev --type=shared`
  - Pushed to git ONLY as a reference name (e.g. in `pigsty.yml.j2`), never the value
- **User-generated-manual**: anything that can't be rotated from a fresh install
  - `KUBE_VAULT_PASSWORD` ŌĆö kubespray cluster-vault, used to decrypt subsequent kubespray runs
  - `PVE_API_TOKEN` ŌĆö must be created in the PVE UI, scoped to a per-purpose user
  - `GARAGE_ROOT_TOKEN` ŌĆö issued by `garage` CLI on first deploy, one-time
  - `ARGOCD_INFISICAL_CLIENT_ID/SECRET` ŌĆö separate Machine Identity in Infisical, scoped to `infra-bootstrap` project only
- **Tool-generated-and-captured**: anything kubespray/pigsty/Garage generates during install
  - We capture the output and write it to Infisical in a post-install hook (manual, scripted)

## Rotation policy

**None yet.** Cluster is being built from scratch. Once live:

- All agent-generated passwords: rotate every 90 days, scripted via `infisical secrets set` + cluster reload
- All user-generated values: rotate manually with the user
- All break-glass tokens: rotate every 30 days OR after any use

## Bootstrap order (what gets created when)

1. User creates `KUBE_VAULT_PASSWORD` + `PVE_API_TOKEN` (offline, before any tool runs)
2. User creates `GARAGE_ROOT_TOKEN` (after Garage LXC first boot)
3. User creates `ARGOCD_INFISICAL_CLIENT_ID/SECRET` (after ArgoCD install)
4. Agent generates all DB passwords and writes to Infisical
5. Kubespray runs ŌåÆ agent captures etcd certs/tokens from stdout, writes to Infisical
6. Pigsty runs ŌåÆ reads via `infisical export`
7. Agent creates `K8S_BREAK_GLASS_TOKEN` after cluster is up
