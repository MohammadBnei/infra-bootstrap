# Secret schema — `infra-bootstrap` Infisical project

The single source of truth for what lives in the `infra-bootstrap` Infisical project
(env `dev`). Referenced by `pigsty.yml` and by kubespray `group_vars/`.

## Layout

```
infra-bootstrap (project)
/k8s                       — kubespray, ArgoCD, K8s SAs
  /argocd                  — ArgoCD → Infisical credentials (machine identity)
  /break-glass             — cluster-admin token for emergency recovery
  /etcd                    — etcd peer certs + initial cluster token
/postgres                  — Pigsty, Patroni, pgBackRest
  /pg-meta                 — current source cluster (192.168.1.193) — read-only reference
  /pg01                    — primary data node (proxmox .165)
  /pg02                    — replica data node (server1 .200)
  /etcd-witness            — etcd quorum witness
  /users                   — dbuser_* passwords (meta, view, infisical, ory, ...)
  /haproxy                 — HAProxy admin stats
  /grafana                 — Grafana admin
  /pgadmin                 — pgAdmin
  /pgbackrest              — pgBackRest S3 backup target (Garage)
/cluster-bootstrap         — one-shot bootstrap material, consumed then discarded
  /pve                     — Proxmox API token used by kubespray/Ansible
  /ssh                     — pre-shared SSH key passphrase (if any)
  /nfs                     — NFS share creds
  /garage                  — Garage S3 root token (used by pgbackrest + apps)
```

## Naming convention

- Secret name: `SCREAMING_SNAKE_CASE`
- Path: `/<component>/[<sub-area>]` — never start a path with `agent-secrets` (out of scope)
- Type: always `shared` (machine identity is project-scoped, no personal secrets)
- Environment: secrets live in `dev`; `prod` is created by duplicating the `dev` tree when the cluster goes live

## Generation rules

- **Agent-generates-random** (32 bytes, base64): all `*_PASSWORD` and `*_TOKEN` rows above
  - Generator: `openssl rand -base64 32` or `head -c 32 /dev/urandom | base64`
  - Written via `infisical secrets set --projectId="$PROJ" --env=dev --path=... --type=shared`
  - Pushed to git ONLY as a reference name (e.g. in `pigsty.yml.j2`), never the value
- **User-generated-manual**: anything that can't be rotated from a fresh install
  - `KUBE_VAULT_PASSWORD` — kubespray cluster-vault, used to decrypt subsequent kubespray runs
  - `PVE_API_TOKEN` — must be created in the PVE UI, scoped to a per-purpose user
  - `GARAGE_ROOT_TOKEN` — issued by `garage` CLI on first deploy, one-time
  - `ARGOCD_INFISICAL_CLIENT_ID/SECRET` — separate Machine Identity in Infisical, scoped to `infra-bootstrap` project only
- **Tool-generated-and-captured**: anything kubespray/pigsty/Garage generates during install
  - We capture the output and write it to Infisical in a post-install hook (manual, scripted)

## Rotation policy

**None yet.** Cluster is being built from scratch. Once live:
- All agent-generated passwords: rotate every 90 days, scripted via `infisical secrets set` + cluster reload
- All user-generated values: rotate manually with the user
- All break-glass tokens: rotate every 30 days OR after any use

## Bootstrap order (what gets created when)

1. User creates `KUBE_VAULT_PASSWORD` + `PVE_API_TOKEN` (offline, before any tool runs)
2. User creates `GARAGE_ROOT_TOKEN` (after Garage LXC first boot)
3. User creates `ARGOCD_INFISICAL_CLIENT_ID/SECRET` (after ArgoCD install)
4. Agent generates all DB passwords in `/postgres/...` and writes to Infisical
5. Kubespray runs → agent captures etcd certs/tokens from stdout, writes to `/k8s/etcd`
6. Pigsty runs → reads from `/postgres/...` via `infisical export`
7. Agent creates `K8S_BREAK_GLASS_TOKEN` after cluster is up
